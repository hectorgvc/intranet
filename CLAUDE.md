## Project Overview

This is an internal intranet web application for **PROMESE/CAL** (Programa de Medicamentos Esenciales y Central de Apoyo a la Logística de Insumos), a Dominican government healthcare agency. It is a client-side SPA (Single Page Application) built with vanilla JavaScript, Tailwind CSS, and Lucide icons.

**Important:** This is a static web application. There is no build process, no npm/node, and no bundler. Simply open `index.html` in a browser or serve with any static file server.

## Development Commands

Since this is a static HTML/JS/CSS application, there are no build or test commands. To run the application:

```bash
# Option 1: Open directly in browser (simplest)
start index.html

# Option 2: Use Python's built-in server (if available)
python -m http.server 8080

# Option 3: Use Node's npx serve (if available)
npx serve .
```

## Architecture Patterns

### State Management (Centralized)
- All state lives in `js/state.js` including default data for portales, directorio, nomenclaturas, and noticias
- State is persisted to `localStorage` automatically via `saveData()`
- On load, state is restored from `localStorage` if available, otherwise uses defaults
- LDAP data takes precedence: `directorio = (typeof LDAP_DATA !== 'undefined') ? [...LDAP_DATA] : [...DEFAULT_DIRECTORIO]`

### SPA Router (Hash-less)
- Navigation uses `navigateTo(viewName, linkElement)` function in `js/router.js`
- Views are `<div>` containers with IDs like `view-inicio`, `view-portales`, etc.
- Only one view has `.active` class at a time (CSS controls visibility)
- Sidebar links get `.active` class for styling
- Each view has a corresponding `renderViewName()` function called on navigation

### Module Organization
- Controllers are feature-based: dashboard.js, directorio.js, nomenclaturas.js, etc.
- Controllers expose `render*()` functions that generate HTML and inject into DOM
- Shared utilities are in `services.js` (toast notifications, CSV export, pagination helpers)
- Each controller file ends with lifecycle hooks being called from router.js

### Data Persistence Strategy
1. **Portales, Noticias, Nomenclaturas**: Admin-managed through UI, stored in localStorage
2. **Directorio**: Synced from Active Directory via PowerShell script (read-only in UI)
3. **Feriados**: Loaded from `js/data/feriados.json` at init time

## LDAP Integration

The directory (directorio) can be synchronized from Active Directory:

- Run `scripts/actualizar_directorio.bat` on a domain-joined Windows machine
- Requires PowerShell and LDAP access to `promese.promesecal.gob.do:389`
- Generates `js/ldap_data.js` which exports `LDAP_DATA` constant
- If LDAP sync fails, falls back to default data in `state.js`

**Note:** The LDAP sync script filters for active users with extensions configured.

## Key Implementation Details

### Calendar System
- Custom calendar in `dashboard.js` with navigation (prev/next month)
- Holidays are fetched from `js/data/feriados.json` and highlighted in red
- Calendar state tracked by `calYear` and `calMonth` variables in state.js

### Email Signature Generator (Firma)
- Generates HTML email signatures matching the institution's official format
- Uses base64-encoded social media icons embedded in the HTML
- Outputs table-based HTML suitable for Outlook
- Copy functionality uses modern Clipboard API with execCommand fallback

### Admin Panel
- Password-protected: `dtic2025`
- Manages news (noticias) displayed on dashboard hero
- Can add/delete directory entries (manual override)
- Other admin modules (nomenclaturas, portales) are stubbed/disabled

### Styling Conventions
- Tailwind CSS via CDN with custom config in `js/config/tailwind.config.js`
- Custom CSS in `css/styles.css` for components not easily done with Tailwind
- Color scheme: Primary `#0f4c5c` (teal), Accent `#e63329` (red)
- Font: Inter (Google Fonts)

## Common Tasks

### Adding a New Portal Link
Edit `DEFAULT_PORTALES` array in `js/state.js`:
```javascript
{ id: N, categoria: 'Category Name', nombre: 'Portal Name', url: 'https://...', descripcion: 'Description', icono: 'lucide-icon-name' }
```

### Updating Holidays
Edit `js/data/feriados.json` with new dates in format `"YYYY-MM-DD"`.

### Modifying Admin Password
Change the hardcoded check in `js/controllers/admin.js` line 6:
```javascript
if (pw === 'dtic2025') { ... }
```

### Adding a New View
1. Add container div in `index.html` with id `view-name`
2. Add sidebar link with `data-view="name"` and `onclick="navigateTo('name', this)"`
3. Create controller file `js/controllers/name.js` with `renderName()` function
4. Add script tag in `index.html`
5. Add lifecycle hook in `js/router.js` `navigateTo()` function
